# Akashi Compliance — Product Value Document

## Что это

Akashi Compliance — автоматизированная система комплаенс-проверки контрагентов для компаний, работающих в Казахстане и с казахстанскими партнёрами.

Система заменяет ручной процесс проверки (2–4 часа на одну компанию) автоматическим пайплайном (3–5 минут), объединяя данные из официальных KZ-реестров (Adata) и международных санкционных баз (LSEG World-Check One).

---

## Для кого

| Роль | Боль | Что решает |
|------|------|------------|
| Compliance Officer | Тратит часы на ручной сбор данных из разных источников | Автоматически собирает всё в одной карточке |
| Compliance Manager | Не видит портфель целиком, нет приоритизации | Дашборд с risk-счётчиками и списком "требуют внимания" |
| Юрист | Читает тексты судебных дел вручную | LLM классифицирует каждое дело: category, severity, summary |
| Директор / Комитет | Нечем обосновать решение перед регулятором | Объяснимый скоринг 0–100 с audit trail по каждой метрике |
| IT / ИБ | Нет интеграции с международными санкциями | LSEG WC1 покрывает OFAC, UN, EU, HMT и 500+ списков |

## Ключевые функции

### 1. Пакетная загрузка контрагентов
Загрузите Excel/Word-файл со списком БИН — система создаёт дела и запускает проверку параллельно для всех компаний. Поддерживаются политики дубликатов: пропустить / обновить / создать новое.

### 2. Двойная проверка: Adata + LSEG World-Check One

**Adata (Казахстан):**
- Регистрационные данные, статус, ОПФ
- Налоговая задолженность и история платежей
- Судебные дела (гражданские, уголовные, административные)
- Аффилированные компании и учредители

**LSEG World-Check One (международный):**
- Санкционные списки OFAC SDN, UN, EU, HMT, SECO и 500+ других
- PEP-скрининг директора и учредителей (политически значимые лица)
- Adverse Media — кураторские статьи: коррупция, мошенничество, терроризм
- Автоматический re-screen существующего портфеля одной кнопкой

### 3. Граф аффилиатов (depth-2)
Система строит граф связанных компаний на 2 уровня вглубь. Выявляет скрытые связи, которые не видны при проверке только самой компании. Интерактивная визуализация с возможностью открыть карточку любого узла.

### 4. Унифицированный скоринг риска (0–100)

| Метрика | Вес | Источник |
|---------|-----|----------|
| Международные санкции | 35 | LSEG WC1 |
| Судебная активность | 20 | Adata + LLM |
| Налоговый комплаенс | 15 | Adata |
| Правовой статус | 15 | Adata |
| PEP-скрининг физлиц | 5 | LSEG WC1 |
| Adverse Media | 5 | LSEG WC1 |
| Риск аффилиатов | 5 | Граф |

Каждый балл сопровождается объяснением (`reason`) и источником (`source`). Решение защищаемо перед регулятором.

### 5. LLM-анализ судебных дел
Каждое судебное дело из Adata автоматически классифицируется языковой моделью:
- `category`: уголовное / гражданское / административное / исполнительное
- `severity`: critical / high / medium / low
- `outcome`: осуждён / в процессе / прекращено
- `summary_ru`: краткое резюме на русском

Уголовные дела с conviction автоматически увеличивают скоринг.

### 6. AI-заключение и чат
После сбора данных система генерирует полное комплаенс-заключение (OpenAI/Groq). Встроенный чат-ассистент отвечает на вопросы по досье: риски, аффилиаты, документы, служебная записка.

### 7. Конструктор отчётов
Выберите нужные разделы (общие сведения, скоринг, LSEG санкции, суды с AI, налоги, аффилиаты, заключение), укажите офицера и уровень конфиденциальности — получите PDF одной кнопкой через браузерную печать.

| Формирование отчёта | Word-шаблон, 30–60 минут | Конструктор, 2 минуты |
| Обоснование решения | Субъективно, слова | Скоринг 0–100, audit trail |

## Уровень автоматизации

```
Загрузка списка БИН
       ↓
Adata enrichment ──────────────────────────┐
       ↓                                   │
LLM-классификация судебных дел             │
       ↓                                   │  параллельно
LSEG WC1 скрининг (компания + директор)   │
       ↓                                   │
Adverse Media check ───────────────────────┘
       ↓
RiskScorer (7 метрик, 0–100)
       ↓
AI-заключение (OpenAI/Groq)
       ↓
Готово: карточка + скоринг + отчёт
```

**~95% процесса автоматизировано** при подключении Adata + LSEG + OpenAI.

Человек принимает финальное решение, опираясь на готовое обоснование.

## Технический стек

| Слой | Технология |
|------|------------|
| API | Python 3.12 + FastAPI |
| База данных | SQLite (→ PostgreSQL в roadmap) |
| Очередь задач | TaskIQ + Redis |
| AI | OpenAI API (совместим с Groq, любым OpenAI-compatible) |
| Данные KZ | Adata API |
| Данные международные | LSEG World-Check One v3 |
| Frontend | Next.js 16 + React 19 + Tailwind 4 |
| Деплой | Docker Compose (4 контейнера) |

## Быстрый старт

```bash
# 1. Клонировать и настроить
cp .env.example .env
# Заполнить ADATA_TOKEN, LSEG_CLIENT_ID, LSEG_CLIENT_SECRET, LSEG_GROUP_ID, OPENAI_API_KEY

# 2. Запустить
docker compose up -d

# 3. Открыть
# Frontend: http://localhost:3001
# API:      http://localhost:8000

# 4. Применить LSEG к существующим данным
curl -X POST http://localhost:8000/api/admin/rescreen
```

## Roadmap

- **Аутентификация** — multi-user, роли (офицер / менеджер / только просмотр)
- **PostgreSQL** — для продуктивных нагрузок и многопользовательского режима
- **Ongoing Screening** — LSEG webhook: автоматический алерт при попадании контрагента в санкционный список
- **Egov / Kompra** — дополнительные KZ-источники
- **Экспорт в Excel** — реестр проверок для внутреннего аудита
